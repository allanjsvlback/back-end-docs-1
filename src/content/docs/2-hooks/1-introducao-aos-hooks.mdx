No React 16.8, a equipe do React introduziu uma adição que mudaria fundamentalmente a forma como escrevemos componentes: os **Hooks**. Eles não são uma funcionalidade opcional ou avançada, mas sim a maneira moderna e recomendada de construir aplicações React.

Para entender por que os Hooks são tão importantes, é útil olhar para o mundo "pré-Hooks".

## O Mundo Antes dos Hooks

Anteriormente, a lógica e o estado de um componente estavam fortemente ligados à sua estrutura de classe. Isso trazia alguns desafios significativos:

<CardGrid>
	<Card title="Componentes de Classe Gigantes">
		Componentes complexos frequentemente se tornavam arquivos enormes e difíceis de manter. Lógicas relacionadas (como buscar dados e se inscrever em eventos) ficavam espalhadas por diferentes métodos de ciclo de vida (`componentDidMount`, `componentDidUpdate`, `componentWillUnmount`).
	</Card>
	<Card title="Lógica Difícil de Reutilizar">
		Reutilizar lógica com estado entre componentes era complicado. Padrões como **Higher-Order Components (HOCs)** e **Render Props** foram criados para resolver isso, mas eles introduziam aninhamento excessivo de componentes (o "wrapper hell") e tornavam o código mais difícil de seguir.
	</Card>
    <Card title="A Complexidade do `this`">
		Componentes de classe exigiam o uso da palavra-chave `this`, uma fonte comum de confusão em JavaScript. Era necessário fazer o `bind` de métodos no construtor, um passo extra que frequentemente era esquecido.
	</Card>
</CardGrid>

## O Que São Hooks?

**Hooks são funções que permitem que você "se conecte" (hook into) aos recursos de estado e ciclo de vida do React a partir de componentes de função.**

Em outras palavras, eles permitem que seus componentes de função, que antes eram simples e "sem estado", façam tudo o que um componente de classe podia fazer, e de uma forma muito mais limpa e direta.

<Aside type="note">
Todos os Hooks começam com a palavra `use`, como `useState` ou `useEffect`. Essa é uma convenção de nomenclatura que o React usa para impor as [Regras dos Hooks](/2-hooks/7-regras-dos-hooks/).
</Aside>

### Os Benefícios dos Hooks

A adoção dos Hooks trouxe uma revolução na escrita de componentes React:

1.  **Código Mais Limpo e Conciso:** Compare um componente de classe com um funcional que usa Hooks. O segundo é quase sempre mais curto, mais fácil de ler e não tem a complexidade do `this`.

2.  **Lógica Agrupada por Assunto:** Com o Hook `useEffect`, você pode agrupar lógicas relacionadas. Por exemplo, o código para buscar dados e o código para "limpar" essa busca (caso o componente seja desmontado) ficam juntos, no mesmo lugar.

3.  **Reutilização de Lógica Elegante:** Hooks permitem extrair lógica com estado para funções reutilizáveis chamadas **Custom Hooks**. Se você tem a mesma lógica de busca de dados em vários componentes, pode criar um hook `useFetch` e usá-lo em qualquer lugar, sem aninhar componentes.

Os Hooks não mudam os conceitos fundamentais do React — componentes, props e state ainda são a base de tudo. Eles apenas fornecem uma API mais direta, poderosa e ergonômica para usar esses conceitos.
